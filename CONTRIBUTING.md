# Contributing to Veil-Browser

Thanks for your interest in contributing!

## Quick Ways to Help

- Star the project
- Report bugs
- Suggest features
- Improve documentation
- Submit code

## Reporting Bugs

Before submitting a bug report:
1. Check if you're using the latest version
2. Search [existing issues](https://github.com/Souvlaki42/Veil-Browser/issues) to avoid duplicates
3. Collect relevant information (OS, version, error messages, steps to reproduce)

**Submit your report:**
- [Open a new issue](https://github.com/Souvlaki42/Veil-Browser/issues/new)
- Describe what you expected vs. what happened
- Include steps to reproduce the bug
- Add relevant screenshots or error logs

> **Security issues:** Email security vulnerabilities privately (email coming soon) instead of creating public issues.

## Suggesting Features

1. Check the [documentation]() and [existing issues](https://github.com/Souvlaki42/Veil-Browser/issues) first
2. Open a new issue with a clear title
3. Explain the feature and why it would benefit users
4. Include mockups or examples if helpful

## Contributing Code

1. Fork the repository
2. Make your changes
3. Write clear commit messages
4. Submit a pull request

### Commit Message Format
- Use present tense ("Add feature" not "Added feature")
- Keep the first line under 50 characters
- Add detailed description if needed

## Code of Conduct

By participating, you agree to uphold our [Code of Conduct](https://github.com/Souvlaki42/Veil-Browser/blob/main/CODE_OF_CONDUCT.md). Please report unacceptable behavior to [email coming soon].

## Questions?

Search [existing issues](https://github.com/Souvlaki42/Veil-Browser/issues) first, then feel free to open a new one if you need help.
